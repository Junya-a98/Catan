# 独立バリアント設計ロードマップ

> **状態:** 第10段階まで完了しました。`standard`、7イベントの `forecast_events`、標準19タイル版と37タイル拡張版の `frontier`、常設市場と公開競売を備えた `trade2`、期限付き借入の `credit`、固定複合 `events_economy_v1` はWeb開始画面から選択できます。第11段階の `grand_campaign_v1` は固定4componentの公開境界とWeb表示まで実装し、サーバー実機回帰と対戦バランスの検証を進めています。

## 目的

通常ルールを安定したまま保ちつつ、対局ごとに異なる遊び方を追加できる境界を作ります。既存の小規模な `HouseRules` とは分離し、開始時に次のいずれか1モードだけを選ぶ構成を想定します。

- `standard`: 現在の通常ルール。バリアント処理を行わない
- `forecast_events`: 数ターン先の全体イベントを予告してから解決する
- `frontier`: 未探索地形を街道の到達に応じて公開する
- `trade2`: 常設市場、競売、条件付き契約を段階的に追加する
- `credit`: 期限付きの資源借入と公開VP減点を追加する
- `composite`: 監査済みの固定catalogだけで複数機能を組み合わせる

単独モードの検証を終えた機能だけを、固定catalogとして段階的に組み合わせます。自由なcomponent配列やモード別optionsは受け付けず、効果の組み合わせ、非公開情報、AI評価をcatalog単位で監査します。

## 通常ルールとの互換境界

予定する設定documentは `kind`、`version`、モード固有の `options` を持ちます。設定がない既存セーブ・リプレイは `standard` として扱います。

```json
{
  "kind": "standard",
  "version": 1,
  "options": {}
}
```

`standard` は無処理の実装にし、現在の盤面生成、資源保存則、合法手、AI、セーブ、リプレイ、LAN / Webスナップショットを変えないことを回帰テストで保証します。バリアント固有の可変状態は通常の `phase` や `house_rules` へ混ぜず、独立した `variant_state` として保存します。

完全なセーブの `variant_state` は `format`、`version`、`kind`、設定の `config_fingerprint`、`public`、`private` を持ちます。LAN / Webスナップショットでは同じ文書から `private` キー自体を除去し、空の秘密領域さえ送信しません。受信側は公開文書の項目、version、kind、fingerprintを `rules.variant` と照合し、観戦履歴の保存境界でも `private` の混入を拒否します。設定または実行時状態がない旧データだけは、空の `standard` として復元します。

対局identityには設定versionとfingerprintを含め、異なるモードのセーブ・リプレイ・通信状態を取り違えないようにします。権威サーバーがすべての効果を解決し、クライアントは公開済みの状態と合法なsemantic commandだけを受け取る方針は維持します。

## `forecast_events`: 予告付き全体イベント

イベントは安定したIDを持つ宣言的なcatalogとして定義し、実行コードやcallbackを保存データへ入れません。イベントを予告する手番と解決する手番は絶対手番番号で記録し、実時間timerには依存させません。

互換catalog `core_v1` は既存セーブ・リプレイ向けに保存形式と並び順を固定し、次の2種類だけを含みます。

- `wheat_harvest_v1`（豊作）: 次に麦が通常生産された後、銀行に対象者全員分の余裕がある場合だけ各対象者へ1枚追加する。追加分を配れない場合も通常生産は取り消さない
- `sheep_drought_v1`（大干ばつ）: 発動から参加人数分の手番が完了するまで、羊タイルの生産を止める

新規対局の既定catalog `core_v2` は、この2種類を含む次の7種類を各1枚ずつ持ち、一部のイベントには予告対象parameterがあります。

| イベント | 予告と効果 | 終了条件 |
| --- | --- | --- |
| 豊作 | 次に麦が通常生産された後、銀行に対象者全員分の余裕があれば各対象者へ麦を1枚追加。追加できなくても通常生産は維持 | 次の麦生産で消費 |
| 大干ばつ | 羊タイルの生産を停止 | 発動から全員が1手番を完了 |
| 港湾封鎖 | 予告された1か所の交換所を使用不可にする。港を使わない銀行交易は可能 | 発動から2手番を完了 |
| 建設ブーム | 全員共通で、次に有料建設される街道1本の木または土1枚を免除。街道建設カードの無料街道では消費しない | 次の有料街道建設で消費 |
| 商人祭 | 成立した国内交易の双方へ、銀行在庫を各1枚ずつ減らしてランダム資源を支給。双方分の在庫がなければ追加配布しない | 発動から全員が1手番を完了 |
| 山賊襲来 | 予告数字の公開タイルから建物による生産力が最も高い場所へ盗賊を即時移動。捨て札と略奪は行わない | 発動時に即時解決 |
| 地震 | 予告方角の辺では新しい街道を建設できず、既存街道も接続と最長交易路の計算に使えない。街道コマは残す | 発動から全員が1手番を完了 |

`core_v2` の対象parameterは権威サーバーが非公開seedから決定し、イベントと同時に事前公開します。港湾封鎖はstableな `harbor_id`、山賊襲来は `5 / 6 / 8 / 9` のいずれか、地震は盤面を6方角に分けたsectorを使用します。保存・通信境界ではイベントIDとparameterの組み合わせを厳密に検証し、別catalogの形式を混ぜた状態は拒否します。

初回は2手番前に予告し、以後は6手番間隔で発動します。絶対完了手番で管理するため、AI速度、再接続、リプレイ再生速度には依存しません。予告・発動・終了をイベントログへ記録し、Webでは次のイベント、残り手番、現在有効な効果を常時確認できます。AIも公開された予告と有効効果だけを判断材料にします。

`core_v1` は従来どおり同じイベントが連続しない4枚の均等なbagを維持します。`core_v2` は7種類を1周期に各1回ずつ発動し、周期境界でも同じイベントが連続しません。未消費の同種効果が次の周期まで残った場合はスタックさせず発動時点だけを更新します。完全セーブ、再接続、リプレイは設定のcatalog ID、対象parameter、進行位置を復元するため、`core_v1` を暗黙に `core_v2` へ変換しません。

未来のイベントデッキと順番は権威サーバーの完全なセーブだけに保持します。LAN / Webの公開スナップショット、観戦、閲覧者別リプレイへは、予告済みイベントと対象、現在有効な効果だけを含めます。予告・発動・終了は重要イベントとして記録し、リザルトから対応するネットワークリプレイへ移動できます。

Web版をバリアントの主導UIとし、広い画面では詳細カード、狭い画面では盤面直前のcompactな予告帯を表示します。封鎖中の交換所と地震中の街道区画は盤面manifestにも状態を持たせ、ラベルだけでなく盤面上でも識別できます。AIは未来のdeckを見ず、公開済みの残り手番と対象だけから資源価値、街道割引、国内交易、港、盗賊、地震区画の評価を調整します。

## `frontier`: フロンティア探索

対局中にタイルや辺を追加するとstable IDが変わり、操作対象、再接続、リプレイが壊れる可能性があります。そのため、全地形と接続関係を対局開始時に事前生成し、一部の地形だけを霧で隠します。

未探索タイルの公開スナップショットでは、資源、数字、港などの秘密を伏せます。街道が探索境界へ到達したときに権威サーバーがタイルを公開し、公開イベントとして保存します。初期配置、生産、盗賊、港、合法な建設先、AI評価は、公開状態を共通の判定関数から参照します。

標準19タイル版では中央7タイルと初期盗賊タイルを公開し、公開領域に接する街道が触れた未探索タイルを公開します。未公開タイルは `UNKNOWN` として資源・数字・盗賊をマスクし、港はmanifestから除外します。ロビー作成時に権威サーバーが提出Seedを秘密Seedへ置き換え、公開ロビー・snapshot・manifestではSeedを0に固定するため、参加者は盤面生成を逆算できません。

37タイル拡張版は明示catalog `outer_ring_37_v1` を使用し、中央砂漠、外周30タイル、96頂点、132辺、12港を対局開始時に事前生成します。中央7タイルだけを初期公開し、標準版と同じ公開ルールで探索します。デスクトップ版はタイル半径を盤面単位で縮小し左右パネルとの余白を保ち、Web版はmanifestの全stable IDを使って同じ霧盤面を描画します。標準19タイル版のoptions、実行時状態、fingerprintは変更せず、既存セーブを暗黙変換しません。

隠れた地形は生産、初期資源、盗賊候補、港交易、建設候補、AIの生産・港評価から除外します。AIは公開情報だけで評価しつつ、未知タイルを開く街道に探索価値を加えます。Webでは生成した霧テクスチャ、公開数カード、公開時アニメーションを表示します。金鉱、古代遺跡、商業都市、危険地帯、中立港はまだ未実装であり、通常の `outer_ring_37_v1` を変更せず、将来の別catalogで追加します。

## `trade2`: 交易2.0

交易2.0は一度に実装せず、次の順に進めます。

1. 資源を `available` / `reserved` / `total` で管理するresource ledger
2. 出品や入札資源を安全に拘束するescrow
3. 手番プレイヤーが出品・購入・取消できる常設市場
4. 手番外の応札権限を備えた公開競売
5. 期限と発動条件を型付きで限定した条件付き契約

resource ledgerとescrowを用意する前に資源を手札から単純に取り除くと、資源カードの保存則、7の捨て札、独占カードなどに不整合や回避手段が生じます。予約済み資源をどの処理で手札として数えるかを先に定義し、銀行・全プレイヤー・escrowを含む総数を検証します。

resource ledger / escrow基盤では従来の `player.resources` を総所持数として保ち、予約だけをversionedなsidecar文書へ保存します。任意の建設・購入・交易は `available` だけを支払えますが、7の閾値と捨て札、略奪、独占は `total` を対象とします。常設市場では手番プレイヤーがAND条件の注文を最大4件まで公開でき、他プレイヤーは自分の行動手番に全量購入できます。注文は既定4手番で期限切れになり、取消・購入・期限切れのいずれでもescrowを原子的に解放または消費します。

強制喪失では対象プレイヤーの市場注文を全件取り消し、escrowを解放してから資源を移します。影響した注文だけを消して失った資源を推測されることを避けるため、公開ログには取消件数だけを残します。注文とledger予約は保存前・復元後に照合し、孤立した注文・予約や改ざんを拒否します。予約文書と次の注文sequenceは完全セーブとauthority rollbackだけに残し、LAN / Web・観戦・ネットワークリプレイの公開snapshotから除去します。本人のWeb表示は総手札と使用可能手札を区別し、相手には従来どおり内訳を公開しません。

公開競売を含む新規catalog `market_auction_v1` は、互換用の `standing_market_v1` を変更せず、常設市場に最大8件の公開競売を追加します。売主は出品資源と最低入札合計枚数を示し、各相手は最大1件の混合資源入札を公開・更新・取消できます。売主は自分の行動手番に任意の入札を選んで落札確定します。売主2件、1競売3入札、既定4完了手番の上限を持ち、期限切れ・取消では全員のescrowを返却します。

`auction_bid` と `auction_cancel_bid` だけを明示的な手番外command allowlistへ入れています。認証済みsessionの席を入札者として使用し、特殊処理・配置選択・ダイス演出中は拒否します。競売revisionと対局全体revisionの両方で競合を検出し、同時入札は先着だけが成立します。AIも公開出品と自分の使用可能手札だけから入札・出品・落札を判断します。

借金制度は `trade2` へ直結せず、後の複合モードでも再利用できる独立domainとして実装します。v1は銀行から指定資源1枚を借り、次の自手番終了までに同種1枚と任意1枚を返済する設計です。期限超過は進行を止めず、任意3枚の延滞債務と公開VP減点へ一度だけ変換します。

独立バリアント `credit` のcatalog `bank_loan_v1` では、1人につき未返済ローンを1件に制限します。通常債務は公開VP−1で、期限内に借りた資源1枚を含む合計2枚を一括返済します。期限を過ぎた時点で一度だけ任意3枚の延滞債務へ変換し、公開VP−2となります。延滞後は任意資源を1枚ずつでも返済でき、未返済を理由に手番進行を止めません。

Webの「資源信用所」は全員の債務状態、残り期限、残債、公開VP減点を表示します。借入・返済の資源内訳を編集できるのは本人だけで、権威サーバーが送る `credit_borrow` / `credit_repay` 候補とローンrevisionに一致するときだけ確定できます。セーブ、再接続、観戦、ネットワークリプレイも同じ公開ローン文書を使用します。

条件付き契約では任意コードや自由記述の式を実行せず、`turn_start` や公開された出目など、有限で検証可能な条件だけを利用します。

## `composite`: 検証済み機能の固定複合

最初の複合catalog `events_economy_v1` は、`forecast_events/core_v2`（2手番前予告・6手番間隔）、`trade2/market_auction_v1`（注文4手番・競売4手番）、`credit/bank_loan_v1` だけを固定で束ねます。盤面は標準19タイルで、`frontier` は含みません。設定documentのoptionsは `{ "catalog": "events_economy_v1" }` だけを許可し、任意のcomponents配列や子optionsによる未検証の組み合わせを拒否します。

実行時状態は外側に共通の `completed_turns` を持ち、3つの子のpublic stateとprivate stateを別々の `components` に格納します。各子は既存の単独 `VariantState` と固定子configで再検証し、すべての子の完了手番が外側と一致しなければ復元しません。LAN / Web投影では外側のprivateと3つの子privateをまとめて除去するため、イベントdeck、注文・競売sequence、ローンsequenceは公開されません。

`component_config(kind)` / `component_state(kind)` は単独モードにも同じ意味で使えます。これにより既存のkind直接比較を段階的に置き換えつつ、1子の同一手番内mutationでは他の子documentを変更しないadapterを利用できます。共通手番を進める処理は3子を一度に更新する必要があるため、Gameのend-turn接続は次の段階で原子的に実装します。

全部入りの `grand_campaign_v1` は、37タイル `frontier`、`forecast_events/campaign_v1`、`trade2/market_auction_v1`、`credit/bank_loan_v1` の4子だけを固定で束ねます。港湾封鎖は予告時点で公開済みの港IDだけを候補として公開planへ固定し、候補がなければ `no_revealed_harbors` としてその回を発動なしにします。予告中に港が公開されても対象を再選択せず、未知の港IDや秘密seedを公開投影へ含めません。Webはこの4つの公開componentと既知catalogだけを読み、既存の `events_economy_v1` とは別の開始項目・ルール説明で表示します。

4. **完了:** 標準19タイル内での `frontier` の霧、秘密Seed、公開処理、Web UI、AI探索評価
5. **完了:** 事前生成した37タイル拡張盤面、表示スケール、AI探索評価、保存・再接続・閲覧者別リプレイ

8. **完了:** 借金制度の独立domain、返済・延滞・公開VP減点、Web UI
9. **完了:** 固定catalog、子config API、厳密な入れ子state、非公開投影を備えた複合モード基盤

各段階でローカル対局、セーブ往復、権威サーバーのrollback、閲覧者別マスキング、リプレイ再現性、AIの合法手を確認します。特に秘密状態は、現在画面だけでなく過去のネットワークリプレイにも漏れないことを検証します。

## 主なリスク

- 予告イベントの効果を増やしすぎることによる組み合わせ爆発
- フロンティアの隠し地形が通信・観戦・リプレイへ漏れること
- 動的な盤面変更によるstable IDの不一致
- escrowと7の捨て札・独占カード間での資源保存則違反
- 手番外入札の二重操作、古いrevision、応答待ちによる進行停止
- AIが公開されていない地形、注文、入札を判断に使用すること

予告イベントとフロンティア拡張はcatalogを分けて互換性と組み合わせを制御しました。`trade2` も `standing_market_v1` を維持したまま `market_auction_v1` へ拡張し、借金制度は独立した `credit` / `bank_loan_v1` として完成させました。標準盤面の `events_economy_v1` に加え、公開済み港だけから港湾封鎖対象を固定する `grand_campaign_v1` の境界、権威実行、保存・再接続・リプレイ、AI、Web表示まで追加しました。次は自己対戦による対局全体のバランス検証と型付き条件付き契約へ進みます。特殊探索物は通常37タイルcatalogへ混在させず、別段階・別catalogで追加します。
