# Catan風ゲーム

Pygame で実装した、ローカル対戦と信頼できるLAN内対戦に対応するカタン風ゲームです。将来のWeb公開に向け、同じ権威サーバーで動くローカル専用ブラウザ版も利用できます。Pygame版は引き続き遊べますが、今後の画面・演出・アクセシビリティ改善はWeb版を主導UIとして進めます。
現在は 2〜4 人プレイ、性格を選べる AI、初期配置、通常ターン、盗賊、銀行交易、プレイヤー間交渉、発展カード、最長交易路 / 最大騎士力まで遊べます。公式基本ゲームに準拠する人数は 3〜4 人で、2 人プレイは簡易バリアントです。

今回の改善で、次の点を優先して強化しています。

- ダイス演出を内部ロジックと同じ `2d6` 表現に統一
- 「今やるべきこと」と「なぜできないか」を UI 上で分かりやすく表示
- 盤面生成に `mode` と `seed` を導入し、再現性と初期盤面バランスを改善
- 開始前 UI から `mode` / `seed` を切り替えて盤面を再生成できるように改善
- 開始前 UI で勝利点を 5〜15 点から選択可能にし、設定をセーブへ保持
- 0〜人数-1人の簡単なAIプレイヤーを追加
- 資源銀行を各19枚にし、配布不足・支払い・捨て札・港交易を公式ルールに合わせて修正
- 街道を木製コマ、開拓地と都市を立体的な建物、盗賊をポーン、港を桟橋と看板として描画
- 港の桟橋を海側へ分離し、港付きの辺に置いた街道が隠れないよう改善
- 港ラベルが右パネルに隠れないよう表示範囲を補正
- プレイヤーごとの記号と駒パターンを加え、色だけに頼らず所有者を判別可能に改善
- 相手の資源内訳と勝利点カードを非公開化
- 人間同士の手番・初期配置・捨て札・交渉の切り替わりに手札を隠す交代画面を追加
- 盤面上部に手番・現在ステップ・次の操作を固定表示
- 左ログは通常折りたたみ、必要な時だけ開いてゲーム開始時まで遡れるよう改善
- AI速度を変更・一時停止でき、判断理由と公開された資源獲得を追いやすく改善
- AIの建設候補、交易、発展カード、盗賊、最大騎士力の評価を強化
- AIの交渉相手を、公開された直近獲得と盤面の生産力から推測して選択
- 国内交易で個別交渉に加えて「全員に募集」を選べるようにし、最初の承諾者と成立
- 対局をイベント単位で自動記録し、終了後や次回起動時に見返せるリプレイビューアーを追加
- 対局後に最終順位・VP推移・累積建設・交易・運指数・重要イベントをまとめるリザルト画面を追加
- リザルトの重要イベントから、対応するリプレイ位置へ直接移動可能
- 同じルール処理でAI戦を反復し、勝率・初期配置・出目を分析する自己対戦ダッシュボードを追加
- 自己対戦をプロセス並列化し、数百〜数千戦の分析を短時間で実行可能に改善
- LAN対戦の権威サーバーとPygameクライアントを追加し、部屋作成・参加コード・再接続・観戦・二重操作防止を実装
- ルール処理と非公開情報マスキングを共用する、依存追加なしのローカル専用ブラウザ版を追加
- ブラウザ盤面を同梱の生成地形・海面、確率ドット、立体的な街道・建物・盗賊・桟橋付き港へ刷新し、外部画像通信なしでローカル版の雰囲気へ統一
- ブラウザの実際の `2d6` 出目を盤面中央で再現し、街道・開拓地・都市の完成時に一度だけ表示する建設アニメーションを追加
- ブラウザへダイス・建設・交易・勝利の効果音と、画面に合わせて曲調が変わるオリジナルBGM、音量・音響ON/OFF設定を追加
- ブラウザへ必要な時だけ開ける建設費・手番・盗賊・交易のルール早見表を追加し、現在のバリアントとハウスルールも明示
- 国内交易の申し込みを専用ポップアップで通知し、「自分が渡す / 受け取る」を固定した条件表示とレスポンシブな調整画面を追加
- 国内交易の受取条件に OR を追加し、「鉄1または麦1」のような代替案から承諾側が実行する1案を選択可能
- 混合AIの実際の性格をロビー・対局・観戦・リプレイでは伏せ、対局後のリザルトだけで公開
- 本人限定の発展カード内訳を、使用候補・今手番購入・秘密勝利点に分けていつでも確認可能
- リプレイ開始時だけ盤面へ移動し、その後の手動・自動再生では閲覧中のスクロール位置を維持
- ブラウザの部屋作成時にAI人数と標準・混合・拡大・交渉・妨害の性格を選択可能にし、AI判断を段階的な盤面更新とコメントで表示
- ブラウザへ最終順位・VP推移・建設・交易・運指数のリザルトと、重要イベントから移動できる閲覧者別リプレイを追加
- ブラウザ通信をWebSocket優先にし、接続できない場合は既存のsame-origin HTTP pollingへ自動でフォールバック
- 予告イベントmodeの既定catalogを `core_v2` へ更新し、対象を事前公開する7イベント、盤面上の封鎖表示、AIの予告対応を追加
- フロンティアmodeへ開始時に全37タイルを事前生成する拡張盤面を追加し、旧19タイル版と明示的に切り替え可能に改善
- Web開始画面へ、予告イベント・常設市場・公開競売・資源信用を同時に遊べる固定複合モード「イベント＆経済」を追加
- 拡張盤面でも未公開30タイルの資源・数字・港・盗賊・Seedを伏せ、保存・再接続・観戦・リプレイでstable IDを維持
- 開始前の詳細設定に、標準19タイルのカスタムマップ編集とハウスルール切り替えを追加
- カスタム設定をセーブ・リプレイ・LANへ同じversion付きdocumentで引き継ぎ、fingerprintで改ざんや取り違えを検出
- AIを標準・拡大重視・交渉重視・妨害重視から選べるようにし、自己対戦では席を自動ローテーション
- 街道建設カードは、合法な配置先がある限り公式どおり2本を連続配置
- 無効ボタンを明確に減光し、残りコマを省略のない4つの情報チップで表示
- セーブ状態を再利用したネットワーク同期プロトコルと、プレイヤー別の非公開情報マスキングを追加
- テストと README を現状仕様に追従

## 起動方法

### 必要環境

- Python 3.10 以上
- `pygame`

### セットアップ

```bash
python3 -m venv .venv
. .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements-dev.txt
```

`Makefile` を使う場合:

```bash
make venv
make install-dev
```

### ゲーム開始

```bash
PYTHONPATH=python python3 python/main.py
```

セットアップ済みなら `make run` でも起動できます。

### ローカルブラウザ版

将来のWeb対戦へ向けたローカル実行版です。追加パッケージは不要です。

```bash
make web
```

起動後に [http://127.0.0.1:8765/](http://127.0.0.1:8765/) を開きます。部屋作成、6文字コードまたは期限付き招待での参加、ready、対局開始、再接続、観戦、サーバーが提示した合法手の操作に対応しています。新規部屋の既定は「期限付き招待リンクのみ」です。ホストはプレイヤー用と観戦用を分けて、1時間・1回限りのリンクを相手ごとに発行できます。待機室では未使用リンクの役割と期限を一覧でき、誤送信したリンクを個別または一括で取り消せます。部屋作成時には総人数の範囲内でAI席を追加し、標準・混合・拡大重視・交渉重視・妨害重視から性格を選べます。AI席は最初からreadyで、サーバーが判断を1段階ずつ進めるため、盤面更新とコメントから行動を追えます。

サーバー再起動後も部屋と進行中の対局へ戻れるようにする場合は、保存先を明示して起動します。

```bash
PYTHONPATH=python python python/web_main.py \
  --state-db saves/web-room-authority.sqlite3 \
  --replay-db saves/web-network-replay.sqlite3 \
  --rate-limit-db saves/web-rate-limits.sqlite3
```

省略時は従来どおりメモリ内だけで動作します。`--state-db` を指定すると、ロビー、開始済み対局、部屋専用乱数、操作sequenceを一括保存し、保存済みの再接続credentialで復帰できます。LAN protocolの再接続tokenは互換性のため発行後も変わりません。WebではtokenをJavaScriptへ返さず、host-only `HttpOnly; SameSite=Strict` 復帰cookieへ直接設定し、復帰成功ごとに新しいtokenへ安全にローテーションします。作成/参加response、WebSocket、bootstrap、`sessionStorage`、`localStorage` に平文tokenは現れません。browserが新cookieを受け取って `POST /api/resume/confirm` で確認するまで、直前に提示したtokenも最大120秒だけ有効です。復帰responseが失われても同じbrowser sessionへ新cookieとbootstrapを再送でき、serverが再起動した場合も猶予内なら旧cookieから復帰を再試行できます。確認後は旧tokenを直ちに失効させ、旧tokenでの再試行を繰り返しても期限は延長しません。

`--replay-db` は `--state-db` と組み合わせた場合だけ有効で、認証済みの各席用・観戦者用ネットワークリプレイを再起動後も復元します。ゲーム本体の保存を先に確定するため、リプレイ保存だけが失敗しても受理済みの操作は取り消しません。欠けたrevisionは再起動境界として明示し、リプレイがゲーム本体より先行する不整合は起動時に拒否します。同じrevisionでも現在の対局と内容が一致しない履歴は、その部屋では利用不可にして古い席別snapshotを返しません。短い参加コードではなく再利用されない部屋IDへ結び付けるため、コードが再利用されても以前の対局履歴は引き継ぎません。省略時はリプレイだけ従来どおりprocess memory内です。

`--rate-limit-db` はWebの操作・入室試行制限を再起動後と同一host上の複数processで共有します。接続元IP、cookie、reconnect token、部屋コードは保存せず、別鍵によるHMAC識別子だけを記録します。3つのDBと鍵は必ずすべて別pathに置き、repositoryへ追加しないでください。認証鍵は既定で各DBの隣に作られ、`--state-key` / `--replay-key` / `--rate-limit-key` で別の安全な場所も指定できます。対局DBとリプレイDBには非公開情報が含まれ、HMACは改ざん検知であって暗号化ではありません。再起動時に接続中だった席の復帰猶予は120秒です。通常のbrowser transport sessionはprocessをまたぎませんが、部屋復帰cookieから同じ席と操作sequenceへ戻れます。

予告イベントmodeでは既定の `core_v2` catalogを使い、次回イベント、対象、発動までの手番数、効果時間、現在有効な効果を表示します。狭い画面では盤面の直前に要点だけの予告帯を表示し、港湾封鎖と地震の対象は盤面にも重ねて示します。予告・発動・終了は履歴へ残り、完全セーブ、再接続、閲覧者別リプレイでも同じcatalogと対象を復元します。未来の順番は権威サーバーだけが保持し、ブラウザ、観戦者、AIへは予告済みの情報だけを公開します。

フロンティア探索は、中央7タイルだけを公開した37タイル拡張版と、従来互換の標準19タイル版を開始画面から選べます。拡張版は `outer_ring_37_v1` として盤面全体、96頂点、132辺、12港のstable IDを開始時に確定し、街道が探索境界へ届いた時だけ隣接タイルを公開します。未公開領域の資源・数字・港・盗賊と盤面Seedは、対局中・観戦・再接続・閲覧者別リプレイのいずれにも送信しません。

画面上部の「ルール」から、建設費、手番の流れ、盗賊、国内・銀行・港交易を必要な時だけ確認できます。ルール早見表には、部屋で有効なバリアントとハウスルールによる例外も表示します。

国内交易を自分が回答する番になると、通常操作より手前に専用ポップアップを表示します。手札を交代して見る段階では条件を伏せ、確認後に「あなたが渡す / あなたが受け取る」の向きを固定して承諾・拒否・条件変更を選べます。条件変更画面も同じ向きで資源ごとの増減と現在条件を表示します。欲しい資源は従来の「すべて」に加え、「鉄1または麦1」のような OR 条件にでき、承諾する側が実行可能な候補を1種類選びます。サーバーが提示した合法な操作だけを有効にし、対戦相手の手札内訳は従来どおり非公開です。

AI性格を「混合」にした場合、CPUごとの割り当てはロビー、対局中、観戦、再接続、リプレイで非公開です。権威サーバー内部では実際の性格で判断し続け、対局終了後のリザルトでだけ公開します。自分のプレイヤーカードでは発展カードの内訳を開閉でき、前の手番までに購入した使用候補、今手番に購入して次手番から使えるカード、秘密の勝利点カードを確認できます。実際に使用できるかは手番やフェーズなどのルールに従って判定されます。

ダイスは権威サーバーが公開した実際の2個の目でアニメーションし、街道・開拓地・都市は新しく置かれた駒だけを強調します。画面上部で効果音とBGMを個別に切り替え、音量を調整できます。BGMはホーム・ロビー・対局・交易で曲調が穏やかに切り替わります。既定はOFFで、ブラウザの自動再生制限に従い、最初の操作より前には音声処理を開始しません。音は外部ファイルを取得せず、このプロジェクト用に作成した音色と短い曲をWeb Audio APIで端末内合成します。

対局中のプレイヤー欄は、建物・最長交易路・最大騎士力と公開中の借入VP減点だけを数えた「公開VP」を表示し、秘密の勝利点カードは本人用の発展カード欄だけで確認できます。対局終了後は全員の真の得点を公開し、開拓地、都市、最長交易路、最大騎士力、勝利点カード、借入減点のVP内訳に加えて、最終順位、VP推移、累積建設数、国内・銀行交易数、運指数、重要イベントを表示します。重要イベントやスライダーから、認証済みの自分用に非公開情報をマスクしたサーバー側リプレイへ移動でき、最初・前・再生 / 一時停止・次・最後・再生速度を操作できます。観戦者のリプレイでは全員の手札内訳を伏せます。

通信はsame-origin WebSocketを優先し、利用できない場合や切断時は既存のJSON APIとHTTP pollingへ自動で切り替わります。複数席を1台で試す場合、cookieを共有しない別ブラウザまたは別プロファイルを使ってください。

友人限定の入口には、既定の期限付き招待を使います。招待URLは `/?room=ABC123#invite=...` 形式で、秘密値をserverへ送る前にbrowser履歴からfragmentを消去します。権威サーバーは平文tokenではなくSHA-256 digest、対象の部屋instance、player / spectatorの権限、有効期限だけを保存し、入室の永続化に成功した時点で一度だけ消費します。同時利用では一人だけが成功し、使用済み・期限切れ・取り消し済み・別部屋・不正tokenは同じ認証エラーとして扱います。ホストの管理画面へ返すのはbearerではない短い管理IDと役割・時刻だけです。Clipboard APIを使えない環境ではWeb Shareへ切り替え、共有も完了しなければ新しい招待を自動で取り消します。発行・一覧・取消・利用は同一端末のloopback、または証明書を検証できるHTTPS / WSSに限定します。

従来どおり、参加コードだけの部屋と、任意の15〜64文字の入室パスフレーズを使う部屋も選べます。参加・観戦時は権威サーバーの認証失敗を受けて専用画面を表示し、パスフレーズは招待URL、ブラウザ保存領域、再接続情報、公開ロビーへ残しません。再接続はserver管理のHttpOnly専用tokenを使うため、パスフレーズや招待tokenの保存・再入力は不要です。部屋作成・参加・退出はcookieを確実に更新できるHTTP経路へ固定し、それ以外の対局操作は引き続きWebSocketを優先します。

既定は従来どおりloopback専用です。離れた友人と試す場合は、Tailscale専用の `--friends-vpn` profileを使えます。このprofileはTLSを必須にし、wildcardではなくホストのTailscale addressだけへbindし、接続元もTailscale device範囲へ限定します。Host / Originは証明書と一致する `.ts.net` 名1つだけを許可し、server側で新規部屋と参加経路を期限付き招待専用に固定します。`--lan` や `--internet-public` との併用、code / passphraseによる直接参加は拒否します。

これはInternetへの直接公開許可ではありません。Tailscale側でも遊ぶ友人とこのportだけをaccess controlで許可し、Funnel、routerのport forwarding、公開IPへのbindは行いません。各参加者へ別々の期限付きリンクを送ります。設定手順と境界は [`docs/friends-vpn.md`](docs/friends-vpn.md)、Web設計とAPIは [`docs/web-mvp.md`](docs/web-mvp.md) を参照してください。SQLite共有rate limitは同一host用であり、一般公開にはaccount認証、永続session、複数hostで共有するbackend、証明書更新などを含む専用profileが必要です。

### AI自己対戦と統計ダッシュボード

通常ゲームと同じルール・AIを、画面表示、音声、待ち時間、リプレイ保存なしで20戦実行します。

```bash
make simulate
```

完了すると `self-play-reports/self-play-latest.json` と、外部通信を行わないローカルHTMLダッシュボードが生成されます。試合数やseedを指定する例:

```bash
PYTHONPATH=python python python/simulate.py --games 100 --seed 42
```

数百〜数千戦をCPU数に合わせて並列実行する例:

```bash
# 0は使用するworker数をCPU数と試合数から自動決定
PYTHONPATH=python python python/simulate.py --games 500 --seed 42 --workers 0

# worker数を固定し、試合ごとの進捗表示を省略
PYTHONPATH=python python python/simulate.py --games 1000 --seed 42 --workers 8 --quiet
```

特定の盤面だけを繰り返し検証する場合:

```bash
PYTHONPATH=python python python/simulate.py \
  --games 100 --board-seed 86712347 --mode constrained
```

4人戦では、指定を省略すると `standard`（標準）、`expansion`（拡大重視）、
`trader`（交渉重視）、`disruptor`（妨害重視）を1人ずつ割り当てます。
性格と席の有利不利を混同しないよう、試合ごとに割り当てる席を1つずつ回します。
構成を固定したい場合は席1から順に指定できます。同じ性格を複数指定するミラー戦も可能です。
以前の全員標準AIによるseed結果を再現するときは、4人戦なら
`--personalities standard,standard,standard,standard` を明示してください。

```bash
PYTHONPATH=python python python/simulate.py --games 100 \
  --personalities expansion,trader,disruptor,standard
```

主なオプション:

- `--games`: 試合数（既定20）
- `--seed`: ダイス・発展カード・略奪を再現する連番seedの開始値
- `--board-seed`: 全試合で固定する盤面seed。省略時は試合ごとに変更
- `--mode`: `constrained` / `fully_random`
- `--players`: 2〜4人（公式基本ゲーム準拠の分析は3〜4人を推奨）
- `--personalities`: 席順のAI性格をカンマ区切りで指定（人数と同数が必要）
- `--target`: 勝利点5〜15
- `--workers`: `0` は自動、`1` は再現性を確認しやすい逐次実行、`2`〜`32` は指定数のプロセスで並列実行（既定 `0`）
- `--open`: 生成後にHTMLを既定ブラウザで開く

各試合にはターン・終了理由の上限を設け、停止した試合を勝率へ混ぜません。さらに資源カード19枚、各プレイヤーの駒数、街道の辺重複、勝者VP、ダイス回数を対局後に検査します。ダッシュボードでは性格別・性格×席の勝率に加え、国内交易の提案・成立、銀行交易、盗賊移動、騎士使用、最終的な街道・開拓地・都市数の1試合平均も比較できます。

AI相性は、性格ごとの全体勝率だけでなく「どの性格を相手にしたとき勝ちやすいか」を相性表で確認します。席順や初手番の偏りも併記されるため、出場数が少ない行や警告付きの数値は結論として扱わず、試合数を増やして比較してください。盤面公平性は `--board-seed` で同じ盤面を固定し、match seedだけを変えて席別勝率・信頼区間を読み、別表の初期配置統計と合わせて判断します。複数の盤面を比較するときは `--board-seed` を変えて同じコマンドを反復します。今回は `--board-count` のような追加APIを設けず、1回のbatchが検証する盤面を明確に保ちます。詳しい結果スキーマは [`docs/self-play-report.md`](docs/self-play-report.md) にあります。

将来のWeb版では、CLIや画面描画に依存しない `game.self_play.run_batch` APIを常駐サーバーのjob queueから呼び出す方針です。HTTPリクエスト内で数千戦を待たせず、job IDで進捗を取得し、完成したversion付きJSON / HTMLを後から参照できる境界にします。

開始前の右パネルで、次の設定をそのまま変更できます。

- プレイヤー人数
- AI プレイヤー人数
  - `0` から `プレイヤー人数 - 1` までボタンで切り替え
- AI 性格
  - 性格ボタンで `標準`（既定）/ `混合` / `拡大重視` / `交渉重視` / `妨害重視` を選択
  - `混合` は CPU ごとに `拡大重視` / `交渉重視` / `妨害重視` のいずれかを非公開でシャッフル
  - ローカル対局は端末内の秘密乱数、Web対局は非公開match seedを使い、実際の性格は対局後にだけ公開
- 勝利点
  - `5` から `15` 点まで増減（既定は `10` 点）
- 盤面生成 `mode`
  - `制約付き` (`constrained`)
  - `公式ランダム` (`fully_random`)
  - 詳細設定で編集した `カスタム` (`custom`)
- 詳細設定
  - 地形（数字チップごと）・数字チップ・港を2つ選んで入れ替え
  - 地形・数字・港を個別にシャッフル
  - 資源18枚＋砂漠1枚、公式数字チップ18枚、港9か所の構成は常に維持
  - 6/8隣接などの偏りは適用前に警告
  - `キャンセル` では盤面を変更せず、`適用` 時にまとめて反映
- ハウスルール
  - 港がない銀行交易を `3:1` に変更（資源港の `2:1` は維持）
  - `7` の出目で手札を捨てず、盗賊移動から開始
  - 発展カードを種類ごとに山札から除外
  - 既定はすべて無効で、従来どおりのルール
- `seed`
  - 直接入力して `seed反映`
  - `再生成` で新しい盤面を作成
- 開始前サマリー
  - 現在の mode / seed
  - mode の簡易説明
  - seed 入力中の操作ガイド

生成盤面は同じ `mode + seed`、カスタム盤面は同じmap documentなら再現できます。
設定は最初の初期ダイスを振るまで変更でき、開始後は対局とリプレイの整合性を保つため固定されます（AI速度は変更可能です）。
カスタム設定のdocument形式とWeb版での再利用方針は [`docs/custom-settings.md`](docs/custom-settings.md) を参照してください。

## LAN対戦

LAN / オンライン対戦の前提として、次の権威サーバー基盤を実装しています。

- セーブ状態を基にしたrevision付き同期スナップショット
- 閲覧プレイヤー本人以外の資源内訳・発展カード・勝利点カードを自動で非公開化
- 発展カード山札は順番を伏せ、残り枚数だけを同期
- TCPの分割受信・複数メッセージ結合に対応するlength-prefix JSON frame
- 最大メッセージサイズ、protocol version、操作種別の検証
- 2〜4人の部屋作成、参加コード、ready、host開始、観戦
- 人間を最低1人残したAI席の予約、AI性格設定、ready不要のCPU参加者表示
- AI判断を上限付きの1ステップごとに進め、各判断を個別revisionとして配信
- 再接続tokenによる席の復帰と、revision / sequenceによる二重操作防止
- opt-in SQLite保存による待機室・開始済み対局・乱数・操作sequenceの再起動復元
- 待機室の明示退出による即時席解放と、host不在時の接続済みplayerへの権限引き継ぎ
- 対局中の再接続期限切れを全参加者へ通知し、進行不能な部屋を安全に終了
- 対局中の明示退出は全参加者へ通知し、その対局を即時終了
- 接続sessionから席を確定するsemantic command検証
- Webでも再利用できるstable ID付き公開盤面manifest
- 閲覧席ごとに非公開情報を伏せた上限付きサーバーリプレイと、公開リザルトJSON
- 開始前画面から開けるレスポンシブロビーと対局画面
- serverが配布する合法手だけを表示・送信する操作UIと、読み取り専用の観戦UI
- Web sessionの接続元紐付け、Host / Origin検証、操作・heartbeatのrate limit
- HMAC匿名化SQLiteによる、再起動・同一host複数process間のrate limit共有
- 非loopback bindと未完成のInternet公開flagを安全側で拒否

開始前画面の `LAN対戦` からロビーを開けます。同じ端末がhostとclientの両方となる部屋作成と、`host:port` を指定する参加・観戦に対応しています。既定のportは `47624` です。

遊び方:

1. ホストは開始前画面で人数・勝利点・盤面を決め、`LAN対戦` → `部屋を作成` を選びます。
2. 参加者は表示された接続先と6文字の参加コードを入力し、`参加する` または `観戦する` を選びます。
3. プレイヤー全員が `準備完了` にした後、ホストが `対局を開始` を押します。
4. 対局中はサーバーが提示した合法なボタンや光る配置先だけを操作できます。`Esc` で接続を保ったままロビー情報へ戻れます。

macOSのファイアウォール確認が出た場合は、同じ信頼できるLAN内で遊ぶときだけPythonへの着信を許可してください。Internetへ `47624` を公開しないでください。

## 盤面生成

開始前 UI に加えて、`GameBoard` と `CatanGame` はコードからも `mode` と `seed` を受け取れます。

- `constrained`
  - デフォルト
  - `6` と `8` の隣接を避ける
  - 高確率トークンの偏りを減らす
- `fully_random`
  - 資源・数字・港をシャッフル
  - 公式ルールに従い `6` / `8` は隣接させない

`seed` を固定すると、同じ資源配置 / 数字配置 / 港配置を再生成できます。

開発用にコードから起動する例:

```python
from game.game import CatanGame

CatanGame(board_mode="fully_random", board_seed=42, ai_player_count=1).run()
```

python3 -m venv .venv
. .venv/bin/activate

PYTHONPATH=python python -m pytest tests

make test

### 独立バリアントのロードマップ

無処理の `standard` に加え、数手番先の全体イベントを公開する `forecast_events` と、霧を探索する `frontier` を実装済みです。Webの部屋作成時に、37タイルの `outer_ring_37_v1` と従来互換の標準19タイル版を切り替えられます。予告イベントの既定catalog `core_v2` には「豊作」「大干ばつ」「港湾封鎖」「建設ブーム」「商人祭」「山賊襲来」「地震」の7種類があり、対象を含む予告・発動・終了をログと公開状態へ記録します。既存セーブ用の `core_v1` は「豊作」「大干ばつ」の構成と保存形式を変更せず、そのcatalog IDのまま読み込みます。

`core_v2` の一時効果は、通常生産への麦追加、1ラウンドの羊生産停止、指定交換所の2手番封鎖、次の有料街道1本の木または土1枚免除、1ラウンド中の国内交易成立ボーナス、指定数字の高生産タイルへの盗賊即時移動、指定方角の街道を1ラウンドだけ接続・最長交易路に使えなくする処理です。山賊襲来では捨て札と略奪は発生せず、地震でも街道コマは失いません。AIは公開済みの予告だけを使い、割引、交易ボーナス、封鎖港、対象数字、地震区画を建設・交易・盗賊判断へ反映します。

フロンティアは中央7タイルだけを公開し、公開領域に接する街道を建設すると隣の霧が開きます。37タイル版は中央砂漠を固定して外周30タイルを隠し、19タイル版は既存セーブと同じ設定document・fingerprintを維持します。未公開タイルは資源・数字・港を通信へ含めず、ロビーと対局snapshotの盤面Seedも伏せます。隠れたタイルは生産せず、隠れた港は利用できません。盗賊・合法手・AI評価も公開領域だけを参照します。

完全なセーブにだけ権威サーバー用の秘密状態を残し、LAN / Web・観戦・ネットワークリプレイの公開snapshotでは `private` キー自体を除去します。予告イベントは対象parameterも検証して保存・再接続・リプレイへ引き継ぎます。外周付きフロンティア盤面と表示スケールに加え、交易2.0の資源台帳・論理エスクロー、常設市場、公開競売まで完了しました。Webでは出品、手番外入札、入札更新・取消、売主による落札確定、残り期限を確認でき、AIも公開条件と自分の使用可能手札だけから判断します。予約済み資源は建設や通常交易へ二重使用できず、7・略奪・独占の直前には市場・競売の予約を安全に自動取消します。

独立モード `credit` / `bank_loan_v1` では、銀行から指定資源1枚を借り、次の自手番終了までに同種1枚と任意1枚を返済できます。未返済中は公開VP−1、期限超過後は任意3枚の延滞債務と公開VP−2へ移行します。延滞は分割返済でき、手番進行を止めません。Webの資源信用所では全員の債務状態を公開し、本人だけが借入・返済内容を調整できます。

固定複合モード `composite` / `events_economy_v1` は、予告イベント、交易2.0（常設市場と公開競売）、資源信用を標準19タイルで同時に有効化します。別モードの `grand_campaign_v1` は37タイル探索も加えた「全部入り」で、Web開始画面から選択できます。Webは固定された公開componentだけを共通adapterから読み、予告、探索、市場、競売、信用を同時表示します。港湾封鎖は予告時点の公開済み交換所だけを候補に固定し、候補がなければ発動なしと明示するため、未知の港を予告から推測できません。特殊探索物は通常の37タイルcatalogへ後付けせず、別catalogとして互換性を保って追加します。詳細は [`docs/variants-roadmap.md`](docs/variants-roadmap.md) を参照してください。
