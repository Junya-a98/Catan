# Catan風ゲーム

Pygame で実装したローカル対戦向けのカタン風ゲームです。
現在は 2〜4 人プレイ、簡単な AI、初期配置、通常ターン、盗賊、銀行交易、プレイヤー間交渉、発展カード、最長交易路 / 最大騎士力まで遊べます。公式基本ゲームに準拠する人数は 3〜4 人で、2 人プレイは簡易バリアントです。

今回の改善で、次の点を優先して強化しています。

- ダイス演出を内部ロジックと同じ `2d6` 表現に統一
- 「今やるべきこと」と「なぜできないか」を UI 上で分かりやすく表示
- 盤面生成に `mode` と `seed` を導入し、再現性と初期盤面バランスを改善
- 開始前 UI から `mode` / `seed` を切り替えて盤面を再生成できるように改善
- 0〜人数-1人の簡単なAIプレイヤーを追加
- 資源銀行を各19枚にし、配布不足・支払い・捨て札・港交易を公式ルールに合わせて修正
- 街道を木製コマ、開拓地と都市を立体的な建物、盗賊をポーン、港を桟橋と看板として描画
- 港ラベルが右パネルに隠れないよう表示範囲を補正
- プレイヤーごとの記号と駒パターンを加え、色だけに頼らず所有者を判別可能に改善
- 相手の資源内訳と勝利点カードを非公開化
- 人間同士の手番・初期配置・捨て札・交渉の切り替わりに手札を隠す交代画面を追加
- 盤面上部に手番・現在ステップ・次の操作を固定表示
- 左ログを手番間で保持し、直前のダイス・建設・交易結果を要約表示
- AIの行動間隔を見直し、行動理由とターン要約を追いやすく改善
- テストと README を現状仕様に追従

## 起動方法

### 必要環境

- Python 3.10 以上
- `pygame`

### セットアップ

```bash
python3 -m venv .venv
. .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements-dev.txt
```

`Makefile` を使う場合:

```bash
make venv
make install-dev
```

### ゲーム開始

```bash
PYTHONPATH=python python3 python/main.py
```

セットアップ済みなら `make run` でも起動できます。

開始前の右パネルで、次の設定をそのまま変更できます。

- プレイヤー人数
- AI プレイヤー人数
  - `0` から `プレイヤー人数 - 1` までボタンで切り替え
- 盤面生成 `mode`
  - `制約付き` (`constrained`)
  - `公式ランダム` (`fully_random`)
- `seed`
  - 直接入力して `seed反映`
  - `再生成` で新しい盤面を作成
- 開始前サマリー
  - 現在の mode / seed
  - mode の簡易説明
  - seed 入力中の操作ガイド

同じ `mode + seed` なら、同じ盤面を再現できます。

## いま遊べる内容

- 初期ダイスで配置順を決定
  - 最高点の同点だけ自動で再ロール
  - 開始プレイヤー以降は席順を維持
- 初期配置 2 ラウンド
  - 2 ラウンド目の開拓地で初期資源を獲得
- 通常ターン
  - `2d6` ダイス演出
  - 出目に応じた資源配布
  - `7` で盗賊フェーズ
- 建設
  - 街道 / 開拓地 / 都市
- 発展カード
  - 騎士 / 街道建設 / 収穫 / 独占 / 勝利点
- 銀行交易
  - 港の交易比率にも対応
- 有限の資源銀行
  - 各資源19枚
  - 複数プレイヤーへの配布不足時は該当資源を配布しない
  - 1人だけが対象なら銀行に残っている枚数まで配布
- 簡単なAI
  - 初期配置 / ダイス / 捨て札 / 盗賊 / 建設 / 発展カード / 銀行交易 / 国内交易を自動処理
- 国内交易・交渉
  - ダイス解決後、建設などと好きな順番で何度でも実行可能
  - 手番プレイヤーから任意の相手へ複数資源・複数枚を提案
  - 承諾 / 拒否 / 条件変更に対応
  - AIは建設目標と資源価値から自動で回答
  - 人間同士では手札を隠す画面受け渡しを表示
- 勝利条件
  - 10 点到達で勝利

## UI の見方

- 盤面上部の進行ヘッダー
  - 「あなたの手番」または「AI進行中」をプレイヤー色で表示
  - `ダイス → 行動 → 終了` の現在位置を常時表示
  - 盗賊や捨て札中は、その特殊処理の指示へ切り替え
- 右パネル
  - 最上部の「今やること」に必要な操作を固定表示
  - 建設ボタンに必要資源を併記
  - 無効な行動は押せない状態、手番終了などの主要操作は強調表示
  - 建設プレビュー = 資源不足 / 在庫切れ / 建設可能 を表示
  - 銀行に残っている各資源の枚数を表示
  - ボタンが多い場面では建設プレビューを2列に圧縮し、下端の見切れを防止
  - 開始前は盤面 `mode` と `seed` をここで調整
- 盤面ハイライト
  - 光っている場所だけが現在の有効候補
  - 盗賊移動、略奪対象、建設候補も状態に応じて強調
- 左ログ
  - 直前のダイス・建設・交易・発展カード結果を上部カードに表示
  - 手番が変わってもイベント履歴を保持
  - 無効操作の理由
  - 資源配布や効果解決の履歴
- プレイヤーカード
  - 現在の手番をプレイヤー色の太枠と「手番中」で表示
  - `あなた` / `人間` / `CPU` を区別
  - `● / ◆ / ▲ / ■` の記号でプレイヤーを色以外でも区別
- 画面上の一時メッセージ
  - クリック直後や無効操作直後に、失敗理由を短く表示
- 数字トークン
  - `6` / `8` は赤で強調
  - 下の pip 数で出やすさを表現

## 操作方法

### 基本操作

- `Space`: ダイスを振る / 初期ダイス
- `Enter`: 手番終了
- `Esc`: 建設や選択をキャンセル
- `H`: 左下のヘルプを開閉

### 建設・行動

- `R`: 街道
- `S`: 開拓地
- `C`: 都市
- `D`: 発展カード購入
- `T`: 銀行交易
- `P`: プレイヤーとの交渉

### 発展カード

- `K`: 騎士
- `B`: 街道建設
- `Y`: 収穫
- `M`: 独占

### 資源選択系

- `1`: 木
- `2`: 羊
- `3`: 麦
- `4`: 土
- `5`: 鉄

## 盤面生成

開始前 UI に加えて、`GameBoard` と `CatanGame` はコードからも `mode` と `seed` を受け取れます。

- `constrained`
  - デフォルト
  - `6` と `8` の隣接を避ける
  - 高確率トークンの偏りを減らす
- `fully_random`
  - 資源・数字・港をシャッフル
  - 公式ルールに従い `6` / `8` は隣接させない

`seed` を固定すると、同じ資源配置 / 数字配置 / 港配置を再生成できます。

開発用にコードから起動する例:

```python
from game.game import CatanGame

CatanGame(board_mode="fully_random", board_seed=42, ai_player_count=1).run()
```

## テスト

依存は `.venv` など、ゲーム起動と同じ Python 環境へ入れてください。

```bash
python3 -m venv .venv
. .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements-dev.txt
```

その上で、次のコマンドを推奨します。

```bash
PYTHONPATH=python python -m pytest tests
```

`Makefile` を使う場合:

```bash
make test
```

`pytest` 単体コマンドを使う場合も、必ず同じ仮想環境を有効化してください。環境が分かれていると `pygame` や `pytest` の片方だけが見つからないことがあります。

CI では GitHub Actions で headless 実行します。

- `SDL_VIDEODRIVER=dummy`
- `SDL_AUDIODRIVER=dummy`
- `PYTHONPATH=python`

ローカルで headless 実行したい場合も同じ環境変数を付ければ CI と近い条件で確認できます。

```bash
SDL_VIDEODRIVER=dummy SDL_AUDIODRIVER=dummy PYTHONPATH=python python -m pytest tests
```

主なテスト対象:

- `2d6` ダイス演出と状態遷移
- 建設候補ハイライト
- 無効操作理由とガイダンス文言
- 開始前 UI の盤面 `mode` / `seed` 設定
- `constrained` 盤面での `6/8` 非隣接
- `seed` 固定時の再現性
- 港交易レート
- 資源銀行の保存則と在庫不足時の配布
- AI の初期配置・建設・ターン進行
- 国内交易の提案・拒否・条件変更・AI応答・資源保存則
- 人間プレイヤー交代時の手札非表示と、AIが非公開の手札内訳を参照しないこと
- 街道・開拓地・都市・港の描画
- 港ラベルの表示範囲
- 最長交易路 / 盗賊の基本処理
