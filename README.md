# Catan風ゲーム

Pygame で実装したローカル対戦向けのカタン風ゲームです。
現在は 2〜4 人プレイ、簡単な AI、初期配置、通常ターン、盗賊、銀行交易、プレイヤー間交渉、発展カード、最長交易路 / 最大騎士力まで遊べます。公式基本ゲームに準拠する人数は 3〜4 人で、2 人プレイは簡易バリアントです。

今回の改善で、次の点を優先して強化しています。

- ダイス演出を内部ロジックと同じ `2d6` 表現に統一
- 「今やるべきこと」と「なぜできないか」を UI 上で分かりやすく表示
- 盤面生成に `mode` と `seed` を導入し、再現性と初期盤面バランスを改善
- 開始前 UI から `mode` / `seed` を切り替えて盤面を再生成できるように改善
- 開始前 UI で勝利点を 5〜15 点から選択可能にし、設定をセーブへ保持
- 0〜人数-1人の簡単なAIプレイヤーを追加
- 資源銀行を各19枚にし、配布不足・支払い・捨て札・港交易を公式ルールに合わせて修正
- 街道を木製コマ、開拓地と都市を立体的な建物、盗賊をポーン、港を桟橋と看板として描画
- 港の桟橋を海側へ分離し、港付きの辺に置いた街道が隠れないよう改善
- 港ラベルが右パネルに隠れないよう表示範囲を補正
- プレイヤーごとの記号と駒パターンを加え、色だけに頼らず所有者を判別可能に改善
- 相手の資源内訳と勝利点カードを非公開化
- 人間同士の手番・初期配置・捨て札・交渉の切り替わりに手札を隠す交代画面を追加
- 盤面上部に手番・現在ステップ・次の操作を固定表示
- 左ログは通常折りたたみ、必要な時だけ開いてゲーム開始時まで遡れるよう改善
- AI速度を変更・一時停止でき、判断理由と公開された資源獲得を追いやすく改善
- AIの建設候補、交易、発展カード、盗賊、最大騎士力の評価を強化
- AIの交渉相手を、公開された直近獲得と盤面の生産力から推測して選択
- 国内交易で個別交渉に加えて「全員に募集」を選べるようにし、最初の承諾者と成立
- 対局をイベント単位で自動記録し、終了後や次回起動時に見返せるリプレイビューアーを追加
- 同じルール処理でAI戦を反復し、勝率・初期配置・出目を分析する自己対戦ダッシュボードを追加
- 街道建設カードは、合法な配置先がある限り公式どおり2本を連続配置
- 無効ボタンを明確に減光し、残りコマを省略のない4つの情報チップで表示
- セーブ状態を再利用したネットワーク同期プロトコルと、プレイヤー別の非公開情報マスキングを追加
- テストと README を現状仕様に追従

## 起動方法

### 必要環境

- Python 3.10 以上
- `pygame`

### セットアップ

```bash
python3 -m venv .venv
. .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements-dev.txt
```

`Makefile` を使う場合:

```bash
make venv
make install-dev
```

### ゲーム開始

```bash
PYTHONPATH=python python3 python/main.py
```

セットアップ済みなら `make run` でも起動できます。

### AI自己対戦と統計ダッシュボード

通常ゲームと同じルール・AIを、画面表示、音声、待ち時間、リプレイ保存なしで20戦実行します。

```bash
make simulate
```

完了すると `self-play-reports/self-play-latest.json` と、外部通信を行わないローカルHTMLダッシュボードが生成されます。試合数やseedを指定する例:

```bash
PYTHONPATH=python python python/simulate.py --games 100 --seed 42
```

特定の盤面だけを繰り返し検証する場合:

```bash
PYTHONPATH=python python python/simulate.py \
  --games 100 --board-seed 86712347 --mode constrained
```

主なオプション:

- `--games`: 試合数（既定20）
- `--seed`: ダイス・発展カード・略奪を再現する連番seedの開始値
- `--board-seed`: 全試合で固定する盤面seed。省略時は試合ごとに変更
- `--mode`: `constrained` / `fully_random`
- `--players`: 2〜4人（公式基本ゲーム準拠の分析は3〜4人を推奨）
- `--target`: 勝利点5〜15
- `--open`: 生成後にHTMLを既定ブラウザで開く

各試合にはターン・終了理由の上限を設け、停止した試合を勝率へ混ぜません。さらに資源カード19枚、各プレイヤーの駒数、街道の辺重複、勝者VP、ダイス回数を対局後に検査します。詳しい結果スキーマは [`docs/self-play-report.md`](docs/self-play-report.md) にあります。

開始前の右パネルで、次の設定をそのまま変更できます。

- プレイヤー人数
- AI プレイヤー人数
  - `0` から `プレイヤー人数 - 1` までボタンで切り替え
- 勝利点
  - `5` から `15` 点まで増減（既定は `10` 点）
- 盤面生成 `mode`
  - `制約付き` (`constrained`)
  - `公式ランダム` (`fully_random`)
- `seed`
  - 直接入力して `seed反映`
  - `再生成` で新しい盤面を作成
- 開始前サマリー
  - 現在の mode / seed
  - mode の簡易説明
  - seed 入力中の操作ガイド

同じ `mode + seed` なら、同じ盤面を再現できます。
設定は最初の初期ダイスを振るまで変更でき、開始後は対局とリプレイの整合性を保つため固定されます（AI速度は変更可能です）。

## いま遊べる内容

- 初期ダイスで配置順を決定
  - 最高点の同点だけ自動で再ロール
  - 開始プレイヤー以降は席順を維持
- 初期配置 2 ラウンド
  - 2 ラウンド目の開拓地で初期資源を獲得
- 通常ターン
  - `2d6` ダイス演出
  - 出目に応じた資源配布
  - `7` で盗賊フェーズ
- 建設
  - 街道 / 開拓地 / 都市
- 発展カード
  - 騎士 / 街道建設 / 収穫 / 独占 / 勝利点
- 銀行交易
  - 港の交易比率にも対応
- 有限の資源銀行
  - 各資源19枚
  - 複数プレイヤーへの配布不足時は該当資源を配布しない
  - 1人だけが対象なら銀行に残っている枚数まで配布
- 簡単なAI
  - 初期配置 / ダイス / 捨て札 / 盗賊 / 建設 / 発展カード / 銀行交易 / 国内交易を自動処理
  - 序盤は資源の多様性、後半は麦・鉄と都市を重視
  - 公開得点の高い相手への盗賊、建設目標を完成させる交易を優先
  - 騎士カードは最大騎士力や盗賊移動の必要がある時まで温存
- 国内交易・交渉
  - ダイス解決後、建設などと好きな順番で何度でも実行可能
  - 手番プレイヤーから任意の相手へ複数資源・複数枚を提案
  - 「全員に募集」では同じ条件を席順に提示し、拒否なら次の相手へ進行
  - 最初に承諾したプレイヤーと交易が成立し、全員拒否なら不成立
  - 承諾 / 拒否 / 条件変更に対応
  - AIは建設目標と資源価値から自動で回答
  - 人間同士では手札を隠す画面受け渡しを表示
- 勝利条件
  - 開始前に設定した 5〜15 点へ到達すると勝利（既定は 10 点）
- セーブ / ロード
  - `F5` でゲーム全体をクイック保存
  - `F9` で保存地点から再開
  - 交渉・盗賊・捨て札・AI設定・全イベント履歴も復元
  - JSON形式、バージョン検査、資源・駒数検証、失敗時ロールバックに対応
- リプレイビューアー
  - 勝利時に対局を `replays/` へ自動保存
  - 終了画面の「対局をリプレイ」、または次回の初期画面の「直前の対局をリプレイ」から開始
  - イベント単位の前後移動、先頭 / 末尾、自動再生 / 一時停止に対応
  - 閲覧中はAI・ダイス・通常入力を完全停止し、終了すると閲覧前の状態へ復帰
  - 通常は公開情報だけを表示し、ローカル分析時だけ `V` で全員の手札を明示公開
  - 過去フレームから対局を再開する機能はなく、閲覧専用
- AI自己対戦・統計ダッシュボード
  - 通常ゲームと同じ `CatanGame` / `SimpleAI` / 公式ルール判定を直接利用
  - match seedと盤面seedを分離し、同じ条件を再現可能
  - 完走率、平均・中央値ターン、席順・初手番・手番位置別勝率を集計
  - 初期配置のpip、6・8への接触、資源種類と勝率を比較
  - `2d6` の実測分布と期待分布の差を表示
  - 全体100戦未満、または比較行20出場未満は小標本として注意を表示
  - JSONと、JavaScript・CDN・外部フォント・アクセス解析を使わないHTMLを原子的に保存

## UI の見方

- 盤面上部の進行ヘッダー
  - 「あなたの手番」または「AI進行中」をプレイヤー色で表示
  - `ダイス → 行動 → 終了` の現在位置を常時表示
  - 盗賊や捨て札中は、その特殊処理の指示へ切り替え
- 右パネル
  - 最上部の「今やること」に必要な操作を固定表示
  - 建設ボタンに必要資源を併記
  - 無効な行動は押せない状態、手番終了などの主要操作は強調表示
  - 手札・街道・開拓地・都市の残数を、完全な名称のチップで表示
  - 建設プレビュー = 資源不足 / 在庫切れ / 建設可能 を表示
  - 銀行に残っている各資源の枚数を表示
  - ボタンが多い場面では建設プレビューを2列に圧縮し、下端の見切れを防止
  - 開始前は盤面 `mode` と `seed` をここで調整
- 盤面ハイライト
  - 光っている場所だけが現在の有効候補
  - 盗賊移動、略奪対象、建設候補も状態に応じて強調
- 左ログ
  - `L` または左上のバーで開閉
  - マウスホイール、`PageUp / PageDown` で移動
  - `Home` でゲーム開始時、`End` で最新へ移動
  - 直前のダイス・建設・交易・発展カード結果を上部カードに表示
  - 手番が変わってもイベント履歴を保持
  - 無効操作の理由
  - 資源配布や効果解決の履歴
- プレイヤーカード
  - 現在の手番をプレイヤー色の太枠と「手番中」で表示
  - 勝利点を `現在点 / 目標点` で常時表示
  - `あなた` / `人間` / `CPU` を区別
  - `● / ◆ / ▲ / ■` の記号でプレイヤーを色以外でも区別
- 画面上の一時メッセージ
  - クリック直後や無効操作直後に、失敗理由を短く表示
- 数字トークン
  - `6` / `8` は赤で強調
  - 下の pip 数で出やすさを表現

## 操作方法

### 基本操作

- `Space`: ダイスを振る / 初期ダイス
- `Enter`: 手番終了
- `Esc`: 建設や選択をキャンセル
- `H`: 左下のヘルプを開閉
- `L`: イベント履歴を開閉
- `A`: AI速度を変更（ゆっくり / 標準 / 高速 / 一時停止）
- `PageUp / PageDown`: 履歴をページ移動
- `Home / End`: 履歴の最初 / 最新へ移動
- `F5`: クイックセーブ
- `F9`: クイックロード

### リプレイ操作

- `← / →`: 1イベント前 / 次へ
- `Home / End`: 最初 / 最後のイベントへ
- `Space`: 自動再生 / 一時停止
- `V`: 全員の手札を公開 / 非公開
- `H`: リプレイ用ヘルプを開閉
- `Esc`: リプレイを終了して元の画面へ戻る

リプレイJSONは手札を含むローカル解析用データです。発展カード山札の順序は保存時に正規化していますが、ファイル自体にはプレイヤーの非公開情報が含まれるため、そのまま他人へ共有しないでください。

セーブ先は `saves/quicksave.json` です。ダイス演出中だけは、出目の二重解決を防ぐため保存できません。

## ネットワーク同期基盤（開発中）

LAN / オンライン対戦の前提として、次の共通プロトコルまで実装しています。

- セーブ状態を基にしたrevision付き同期スナップショット
- 閲覧プレイヤー本人以外の資源内訳・発展カード・勝利点カードを自動で非公開化
- 発展カード山札は順番を伏せ、残り枚数だけを同期
- TCPの分割受信・複数メッセージ結合に対応するlength-prefix JSON frame
- 最大メッセージサイズ、protocol version、操作種別の検証

現時点では通信形式と情報保護までで、ロビー・接続認証・権威サーバーによる操作実行は次の段階です。

### 建設・行動

- `R`: 街道
- `S`: 開拓地
- `C`: 都市
- `D`: 発展カード購入
- `T`: 銀行交易
- `P`: プレイヤーとの交渉

### 発展カード

- `K`: 騎士
- `B`: 街道建設
- `Y`: 収穫
- `M`: 独占

### 資源選択系

- `1`: 木
- `2`: 羊
- `3`: 麦
- `4`: 土
- `5`: 鉄

## 盤面生成

開始前 UI に加えて、`GameBoard` と `CatanGame` はコードからも `mode` と `seed` を受け取れます。

- `constrained`
  - デフォルト
  - `6` と `8` の隣接を避ける
  - 高確率トークンの偏りを減らす
- `fully_random`
  - 資源・数字・港をシャッフル
  - 公式ルールに従い `6` / `8` は隣接させない

`seed` を固定すると、同じ資源配置 / 数字配置 / 港配置を再生成できます。

開発用にコードから起動する例:

```python
from game.game import CatanGame

CatanGame(board_mode="fully_random", board_seed=42, ai_player_count=1).run()
```

## テスト

依存は `.venv` など、ゲーム起動と同じ Python 環境へ入れてください。

```bash
python3 -m venv .venv
. .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements-dev.txt
```

その上で、次のコマンドを推奨します。

```bash
PYTHONPATH=python python -m pytest tests
```

`Makefile` を使う場合:

```bash
make test
```

`pytest` 単体コマンドを使う場合も、必ず同じ仮想環境を有効化してください。環境が分かれていると `pygame` や `pytest` の片方だけが見つからないことがあります。

CI では GitHub Actions で headless 実行します。

- `SDL_VIDEODRIVER=dummy`
- `SDL_AUDIODRIVER=dummy`
- `PYTHONPATH=python`

ローカルで headless 実行したい場合も同じ環境変数を付ければ CI と近い条件で確認できます。

```bash
SDL_VIDEODRIVER=dummy SDL_AUDIODRIVER=dummy PYTHONPATH=python python -m pytest tests
```
